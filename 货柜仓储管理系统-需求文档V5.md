 # 货柜仓储管理系统 - 需求梳理文档（V5 - 终版）
 
 ---
 
 ## 一、系统概述
 
 - **项目名称**：货柜仓储管理系统（第一期）
 - **核心场景**：管理仓库 → 配货 → 智能贩卖机的完整货物流转链路
 - **终端适配**：手机端 + PC 端（响应式或双端适配）
 - **登录权限**：暂不考虑，后续迭代再加
 
 ---
 
 ## 二、模块总览（共 11 个模块）
 
 ```
 商品管理 / 商品规格管理 / 商品分类管理（基础数据）
     │
 仓库管理 ──→ 进货日志 ──→ 仓库库存
                 │
                 ▼
          配货人员 / 送货人员 / 车辆
                 │
                 ▼
          配货记录（配货 → 送货 → 自动更新贩卖机库存）
                 │
                 ▼
          贩卖机管理 ──→ 贩卖机商品（自动同步销售数据）
                          │
                          ▼
                    补货建议 / 每日销售统计
 ```
 
 ---

 ### 模块 5：进货日志管理
 
 **一个进货单可包含多条品类明细**，每条明细关联「商品管理」中的具体商品。
 
 **进货单主表**：
 
 | 字段 | 说明 |
 |---|---|
 | 所属仓库 | 关联仓库 |
 | 入库时间 | |
 | 送货人 | |
 | 收货人 | |
 | 总件数 | 根据明细自动汇总 |
 
 **进货明细（子表）**：
 
 | 字段 | 说明 |
 |---|---|
 | 关联商品 | 选择商品管理中的商品 |
 | 进货数量 | 该品类的进货数量 |
 
 **核心逻辑**：
 
 - **「一键更新库存」按钮**：点击后将本单所有明细的进货数量，按商品分别累加到对应仓库库存中，**每个进货单只能执行一次**，执行后按钮置灰并标记「已入库」
 - **「新增商品」按钮**：进货单内可直接新建商品（自动同步到商品管理）
 - 进货日志支持按仓库、时间筛选查看历史

 ### 模块 8：贩卖机商品管理（核心模块）
 
 **入口**：点击某台贩卖机 → 进入机器详情页
 
 每台贩卖机的商品**关联商品管理表**，获取商品基础信息（名称、价格、规格等），贩卖机层面维护自己的库存、配货数量等运营数据。
 
 一台贩卖机的商品可以来自多个仓库配货，不做仓库限制。
 
 **机器详情页展示**：
 
 | 信息 | 说明 |
 |---|---|
 | 商品名称 | 来自商品管理 |
 | 商品规格 | 来自商品管理 |
 | 商品价格 | 来自商品管理（售卖价格） |
 | 库存 | 该机器当前库存（接口同步） |
 | 配货数量 | 运营设定的目标库存，可调整 |
 | 是否需要配货 | 系统自动判断 |
 | 补货数量 | = 配货数量 - 当前库存（仅需补货时展示） |
 
 **核心逻辑**：
 
 **「是否需要配货」自动判断**：
 ```
 当 商品库存 < 配货数量 × 配货百分比 时
   → 标记为「需要补货」
 配货百分比默认 50%，后台可配置
 ```
 
 **「补货建议」按钮**：
 - 点击后生成该机器的补货清单
 - 补货数量**严格按公式计算**：配货数量 - 当前库存，不可手动调整
 - 筛选出所有「需要补货」的商品
 - 列出商品名称 + 补货数量
 
 **「库存消耗百分比」展示**：
 - 每个商品展示：当前库存 / 配货数量 的百分比
 - 全机器汇总展示：商品总库存 / 商品配货总量 的百分比

 ### 模块 9：每日销售统计
 
 独立的销售数据记录表，从贩卖机接口定时拉取，从系统上线开始记录。
 
 **销售记录表**：
 
 | 字段 | 说明 |
 |---|---|
 | 关联贩卖机 | |
 | 统计日期 | 精确到天 |
 | 当日售出总额 | 该机器当日销售金额 |
 
 **核心逻辑**：
 - 定时任务每天自动从贩卖机接口拉取前一天的销售数据
 - 每台机器每天生成一条销售记录
 - 支持按机器、日期范围查询历史销售数据
 - 展示每日售出总额

 ### 模块 10：配货记录管理
 
 **入口**：配货人员查看「补货建议」或「我的待配货记录」→ 创建配货单
 
 **配货人员和送货人员可以是同一个人。**
 
 **配货单主表**：
 
 | 字段 | 说明 |
 |---|---|
 | 关联贩卖机 | 选择哪台机器需要配货 |
 | 配货人员 | 选择配货人员 |
 | 送货人员 | 选择送货人员（可与配货人员相同） |
 | 使用车辆 | 绑定车辆（关联车辆管理） |
 | 配货完成时间 | 配货人员点击「更新配货」时自动记录 |
 | 送货开始时间 | 送货人员开始配送时记录 |
 | 送货结束时间 | 送货人员点击「完成配送」时自动记录 |
 | 状态 | 待配货 → 配货完成 → 送货中 → 送货完成 |
 
 **配货明细（子表）**：
 
 | 字段 | 说明 |
 |---|---|
 | 关联商品 | 来自商品管理 |
 | 配货数量 | 初始值 = 补货建议公式计算结果，配货人员可自行编辑调整 |
 
 **核心流程**：
 ```
 1. 配货人员查看贩卖机补货建议
 2. 创建配货单（商品和数量自动带入，可手动调整配货数量）
 3. 配货完成 → 点击「更新配货」
    → 自动记录配货完成时间
 4. 送货人员开始送货 → 记录送货开始时间
 5. 送货完成 → 点击「完成配送」
    → 弹出确认提示：「请确认库存数量是否正确，系统将根据配货数量更新贩卖机库存」
    → 确认后：
      · 自动记录送货结束时间
      · 自动更新贩卖机对应商品库存（按配货数量累加）
      · 状态变为「送货完成」
 ```
 
 **个人视角**：配货人员登录后可查看「我的待配货记录」

 ## 六、关键约束汇总
 
 | 约束项 | 规则 |
 |---|---|
 | 商品唯一性 | 商品名称 + 商品规格 不可重复 |
 | 仓库库存唯一性 | 同一仓库 + 同一商品（名称+规格）不可重复 |
 | 进货一键入库 | 每个进货单只能执行一次 |
 | 补货建议公式 | 配货数量 - 当前库存，不可调整 |
 | 配货单数量 | 默认公式值，配货人员可编辑 |
 | 送货确认更新库存 | 弹出确认提示后按配货数量累加到贩卖机库存 |
 | 贩卖机故障 | 不暂停补货建议 |
 | 标签 | 仅用于筛选，不做其他逻辑 |
