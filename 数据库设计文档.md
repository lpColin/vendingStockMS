 # 货柜仓储管理系统 - 数据库设计文档（V1）
 
 ---
 
 ## 一、设计概述
 
 - **数据库**：MySQL 8.0+（建议）
 - **字符集**：`utf8mb4`，排序规则 `utf8mb4_general_ci`
 - **表数量**：17 张业务表 + 1 张系统配置表
 - **命名规范**：表名小写下划线，字段名小写下划线，主键统一 `id`，外键命名 `xxx_id`
 
 ---

 ### 1. product_category（商品分类）
 
 ```sql
 CREATE TABLE product_category (
   id BIGINT PRIMARY KEY AUTO_INCREMENT,
   name VARCHAR(50) NOT NULL COMMENT '分类名称',
   sort_order INT DEFAULT 0 COMMENT '排序',
   created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
   updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
 ) COMMENT='商品分类';
 ```

### 4. product（商品 - 基础数据）
 
 ```sql
 CREATE TABLE product (
   id BIGINT PRIMARY KEY AUTO_INCREMENT,
   name VARCHAR(100) NOT NULL COMMENT '商品名称',
   spec_id BIGINT NOT NULL COMMENT '关联商品规格',
  brand_id BIGINT NOT NULL COMMENT '关联品牌',
   category_id BIGINT NOT NULL COMMENT '关联商品分类',
   price DECIMAL(10,2) NOT NULL COMMENT '商品价格',
   weight INT DEFAULT NULL COMMENT '重量(g)',
   image_url VARCHAR(500) DEFAULT NULL COMMENT '商品图片URL',
   created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
   updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
   UNIQUE KEY uk_name_spec_brand (name, spec_id, brand_id),
   KEY idx_category (category_id),
  KEY idx_spec (spec_id),
  KEY idx_brand (brand_id),
   CONSTRAINT fk_product_spec FOREIGN KEY (spec_id) REFERENCES product_spec(id),
   CONSTRAINT fk_product_category FOREIGN KEY (category_id) REFERENCES product_category(id),
  CONSTRAINT fk_product_brand FOREIGN KEY (brand_id) REFERENCES product_brand(id)
 ) COMMENT='商品';
 ```

 ### 17. product_raw（商品原始数据 - 贩卖机接口）

> 贩卖机接口返回的原始商品数据，用于数据同步和映射到系统商品表。

| 字段 | 类型 | 说明 |
|---|---|---|
| id | BIGINT PK AUTO_INCREMENT | 主键 |
| sku_id | INT NOT NULL | 贩卖机系统SKU编号 |
| sku_name | VARCHAR(200) NOT NULL | SKU名称（如"可口可乐330ml"） |
| base_unit | VARCHAR(10) | 基本单位（如"个"） |
| box_unit | VARCHAR(10) | 箱单位（如"箱"） |
| box_quantity | INT | 每箱数量（如24） |
| spec_str | VARCHAR(50) | 规格描述（如"24个/箱"） |
| image_url | VARCHAR(500) | 商品图片URL |
| box_price | DECIMAL(10,2) | 整箱价格 |
| unit_price | DECIMAL(10,4) | 单个价格 |
| hot | TINYINT DEFAULT 0 | 是否热销：0-否 1-是 |
| package_num | INT | 包装数量 |
| is_recently_purchased | TINYINT DEFAULT 0 | 是否近期购买过：0-否 1-是 |
| stock | INT DEFAULT 0 | 当前库存 |
| stock_r | INT DEFAULT 0 | 库存状态值R |
| stock_p | INT DEFAULT 0 | 库存上限值P |
| stock_a | INT DEFAULT 0 | 累计库存值A |
| stock_b | INT DEFAULT 0 | 库存状态值B |
| stock_w | INT DEFAULT 0 | 库存权重值W |
| brand_id_raw | INT | 贩卖机系统品牌ID |
| brand_name | VARCHAR(100) | 品牌名称 |
| category_code_1 | INT | 一级分类编码 |
| category_name_1 | VARCHAR(50) | 一级分类名称（如"饮料"） |
| category_code_2 | INT | 二级分类编码 |
| category_name_2 | VARCHAR(50) | 二级分类名称（如"碳酸类"） |
| height | DECIMAL(8,2) | 商品高度(cm) |
| length | DECIMAL(8,2) | 商品长度(cm) |
| width | DECIMAL(8,2) | 商品宽度(cm) |
| created_at | DATETIME DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE | 更新时间 |

`sql
CREATE TABLE product_raw (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  sku_id INT NOT NULL COMMENT '贩卖机系统SKU编号',
  sku_name VARCHAR(200) NOT NULL COMMENT 'SKU名称',
  base_unit VARCHAR(10) DEFAULT NULL COMMENT '基本单位(个)',
  box_unit VARCHAR(10) DEFAULT NULL COMMENT '箱单位(箱)',
  box_quantity INT DEFAULT NULL COMMENT '每箱数量',
  spec_str VARCHAR(50) DEFAULT NULL COMMENT '规格描述(24个/箱)',
  image_url VARCHAR(500) DEFAULT NULL COMMENT '商品图片URL',
  box_price DECIMAL(10,2) DEFAULT NULL COMMENT '整箱价格',
  unit_price DECIMAL(10,4) DEFAULT NULL COMMENT '单个价格',
  hot TINYINT DEFAULT 0 COMMENT '是否热销:0否 1是',
  package_num INT DEFAULT NULL COMMENT '包装数量',
  is_recently_purchased TINYINT DEFAULT 0 COMMENT '是否近期购买:0否 1是',
  stock INT DEFAULT 0 COMMENT '当前库存',
  stock_r INT DEFAULT 0 COMMENT '库存状态值R',
  stock_p INT DEFAULT 0 COMMENT '库存上限值P',
  stock_a INT DEFAULT 0 COMMENT '累计库存值A',
  stock_b INT DEFAULT 0 COMMENT '库存状态值B',
  stock_w INT DEFAULT 0 COMMENT '库存权重值W',
  brand_id_raw INT DEFAULT NULL COMMENT '贩卖机系统品牌ID',
  brand_name VARCHAR(100) DEFAULT NULL COMMENT '品牌名称',
  category_code_1 INT DEFAULT NULL COMMENT '一级分类编码',
  category_name_1 VARCHAR(50) DEFAULT NULL COMMENT '一级分类名称',
  category_code_2 INT DEFAULT NULL COMMENT '二级分类编码',
  category_name_2 VARCHAR(50) DEFAULT NULL COMMENT '二级分类名称',
  height DECIMAL(8,2) DEFAULT NULL COMMENT '高度(cm)',
  length DECIMAL(8,2) DEFAULT NULL COMMENT '长度(cm)',
  width DECIMAL(8,2) DEFAULT NULL COMMENT '宽度(cm)',
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  UNIQUE KEY uk_sku_id (sku_id),
  KEY idx_brand (brand_id_raw),
  KEY idx_category1 (category_code_1),
  KEY idx_category2 (category_code_2)
) COMMENT='商品原始数据(贩卖机接口)';
`

> **说明**：
> - 该表存储贩卖机接口返回的原始数据，作为数据同步的中间层
> - sku_id 为贩卖机系统的唯一标识，与系统内部商品表的 id 不同
> - 部分字段（如 stock_r、stock_p 等）为贩卖机内部状态值，具体含义需参考贩卖机接口文档
> - 定时任务从贩卖机接口拉取数据后写入此表，再由业务逻辑映射到系统商品表

 ## 五、补充说明
 
 | 项目 | 说明 |
 |---|---|
 | 标签存储 | `warehouse_stock.tags` 用 JSON 字符串存储，如 `["热销","临期"]`，查询时用 JSON 函数处理 |
 | 进货单号/配货单号 | 业务编号，建议用时间戳+序号生成，如 `PO20260712001`、`DO20260712001` |
 | 售卖价格 | `machine_product.sell_price` 独立于 `product.price`，允许贩卖机层面单独定价 |
 | 库存更新 | 送货完成后按 `delivery_order_item.quantity` 累加到 `machine_product.stock_quantity` |
 | 软删除 | 暂未设计 `deleted_at` 字段，第一期可不考虑，后续按需加 |
 | 重量单位 | `product.weight` 单位为克(g)，INT 类型存储，便于精确计算 |
